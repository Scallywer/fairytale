## Root cause

Stories are ordered by **creation date** in the DB. The `update-image-links.ts` script:

1. **Name-based**: Matches image filename (e.g. `Mala_vila.png`) to story title (e.g. "Mala vila"). Numbered files like `31_Pirgo.png` match "Pirgo" after stripping the prefix.
2. **Order-based**: For stories that didn’t get a name match, it assigns **remaining** images by position using a fixed `storyOrder` list.

Because many Disney/retold stories (Pinokio, Dumbo, Moana, etc.) don’t have exact filename matches (their images are `1_Pinocchio.png`, `2_Dumbo.png`, …), they were filled by **order**. The order-based step uses `storyOrder` and the list of “remaining” images (after name-based matches). So the first unassigned story in `storyOrder` got the first remaining image file, which led to **wrong** pairings: e.g. “Zvijezda iznad Zagreba” got `Nebeski_zmaj.png` (which belongs to “Nebeski zmaj”), and many classic tales got each other’s images.

## How to fix

1. **Re-run `update-image-links.ts`**  
   Ensure it runs after any change so that:
   - Numbered images like `1_Pinocchio.png`, `2_Dumbo.png`, … are matched by **name** (strip number prefix and match to story title, e.g. “Pinokio” vs “Pinocchio” may need a mapping).
   - The `storyOrder` list matches the actual intended story order and only “leftover” images are assigned by position.

2. **Improve name matching**  
   Add handling for:
   - Numbered filenames: `1_Pinocchio.png` → story “Pinokio” (and similar variants).
   - Minor spelling differences (Pinocchio vs Pinokio, etc.) so Disney/retold stories get their correct image by name instead of by order.

3. **Add image for “Mudra baka i vuk”**  
   Either add a file that matches that title or assign an image manually in the DB.

4. **Re-check after fixes**  
   Run `npx ts-node scripts/investigate-image-mapping.ts` again and confirm no WRONG/MISMATCH and only one MISSING (or none once the image is added).
